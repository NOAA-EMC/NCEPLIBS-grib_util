4 C> Program reads selected GRIB2 fields from a file, adds a TOC
5 C> Flag Field separator block and WMO Header in front of each GRIB2
6 C> field, and writes them out to a new file. The output file
7 C> is in the format required for TOC's FTP Input Service, which
8 C> can be used to disseminate the GRIB2 bulletins.
9 C> This service is described at http://weather.gov/tg/ftpingest.html.
10 C>
11 C> PROGRAM HISTORY LOG:
12 C> - 2004-05-17 Gilbert
13 C> - 2012-06-14 VUONG Modified write statement with format specifier
14 C> - 2012-10-22 VUONG CHANGED VARIABLE ENVVAR TO CHARACTER*6
15 C> - 2016-10-15 VUONG Increased length of file name to 200 Characters
16 C>
17 C> USAGE:
18 C> INPUT FILES:
19 C> - 5 - NAMELIST OF GRIB FIELDS AND ASSOCIATED WMO HEADERS.
20 C> - 11 - INPUT GRIB2 FILE.
21 C> - 31 - CORRESPONDING INPUT GRIB2 INDEX FILE.
22 C>
23 C> OUTPUT FILES: (INCLUDING SCRATCH FILES)
24 C> - 6 - STANDARD FORTRAN PRINT FILE
25 C> - 51 - OUTPUT GRIB BULLETIN FILE IN TOC FORMAT
26 C>
27 C> SUBPROGRAMS CALLED: (LIST ALL CALLED FROM ANYWHERE IN CODES)
28 C> UNIQUE: - MAKWMO

30 C> - W3LIB - W3TAGB W3TAGE mkfldsep makwmo

33 C>
34 C> EXIT STATES:
35 C> - COND = 0 - SUCCESSFUL RUN
36 C> - 10 - Error opening input GRIB2 data file
37 C> - 20 - Error opening output GRIB transmission file
38 C> - 19 - ERROR READING CONTROL CARD FILE - All Bulletins missing
39 C> - 30 - Some BULLETINS ARE MISSING
40 C>
41 C> REMARKS: The "EXTRACT" variable in the namelist allows users to choose
